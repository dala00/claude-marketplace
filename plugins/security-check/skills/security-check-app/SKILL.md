description: モバイル/デスクトップアプリのセキュリティチェックを実行する
---

このプロジェクトのモバイル/デスクトップアプリコードに対してセキュリティ監査を実施してください。以下の項目を確認すること：

- **ローカルデータの保護**: ローカルDB（SQLite、Realm など）やファイルの暗号化、Keychain/Keystore の適切な使用
- **証明書ピンニング**: SSL/TLS通信における証明書ピンニングの実装状況
- **ハードコードされた秘密情報**: ソースコードに埋め込まれたAPIキー、トークン、認証情報がないかスキャン
- **依存関係の脆弱性**: パッケージ（pubspec.yaml、Podfile、build.gradle、package.json など）に既知のCVEがないか確認
- **プロセス間通信（IPC）**: Intent、URL Scheme、ディープリンクの安全な処理
- **ログ出力**: デバッグログに機密情報が出力されていないか確認
- **バイナリ保護**: ProGuard/R8 の適用、デバッグビルドの本番リリース防止
- **権限管理**: 不要なパーミッション要求がないか、最小権限の原則に従っているか確認
- **安全でないデータ転送**: HTTP通信（非HTTPS）の使用、App Transport Security の設定

検出結果は重大度レベル（Critical、High、Medium、Low、Info）で報告し、各問題に対して具体的な修正方法を提示すること。
